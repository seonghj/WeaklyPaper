## 웹 서버(Web Server)와 WAS(Web Application Server)의 차이를 설명하고, Spring Boot의 내장 톰캣이 이 둘 중 어디에 해당하는지 설명해주세요.
1. 웹 서버
- HTTP 프로토콜을 통해 클라이언트에게 요청을 받아 Static contents를 제공하는 서버, 프로그램
    1) Static contents (정적 콘텐츠)
        - 서버에 저장된 파일(HTML, JS, CSS, 이미지, 동영상)을 그대로 읽어서 그대로 보내주는 것
        - 모든 사용자들에게 항상 동일한 결과 반환

    2) 특징
        - 구조가 단순함
        - 정적 콘텐츠를 빠르게 전송
        - 코드를 실행할 수 없음

2. WAS
- 동적인 컨텐츠를 처리하고 제공하는 서버
    1) Dynamic contents (동적 콘텐츠)
        - 누가, 무엇을, 어떻게 요청하는지에 따라 각각 다른 응답을 보냄
        - 사용자의 정보에 기반하여 유동적인 결과를 반환

    2) 특징
        - 웹 서버 + 웹 컨테이너(JSP, Servlet)를 합친 형태
        - 코드 실행이 가능하고 DB와 연동 가능
        - 웹 서버의 기능도 수행하면서 동적 콘텐츠 제공 가능

3. Spring Boot의 내장 톰캣은? = WAS
    1) 비지니스 로직을 처리함
        - 개발자가 작성한 로직을 실행 가능
        - 컨트롤러와 서비스/레포지토리를 실행하고 DB와 연동하여 결과를 반환함

    2) 서블릿 컨테이너(Catalina)를 가지고 있음
        - 클라이언트의 요청을 처리하는 서블릿을 관리
        - 멀티 스레딩 가능 (요청마다 스레드 할당)

## Spring Boot에서 사용되는 다양한 Bean 등록 방법들에 대해 설명하고, 각각의 장단점을 비교하세요.
1. @Component 계열 어노테이션
    - 컴포넌트 스캔을 통해 Spring이 직접 찾아 자동으로 Bean을 등록하는 방식

    1) 장점
        - 어노테이션 하나로 등록이 끝남
        - @Component를 상속한 어노테이션(@Service, @Repository, @Controller...)로 역할을 알 수 있어 가독성이 향상
        - JavaConfig 없이도 쉽고 빠르게 설정 가능

    2) 단점
        - Bean 이름, 스코프 등을 세밀하게 설정하기 어려움
        - 자동으로 등록되기 때문에 개발자가 어떤 Bean이 등록되는지 확인하기 어려움
        - 스캔 범위 밖이면 등록 안 됨

2. @Configuration + @Bean
    - 개발자가 직접 Bean 등록용 클래스를 만들어서 Bean을 정의
    - @Configuration 어노테이션을 붙인 클래스 내부에 @Bean 어노테이션이 붙은 메서드를 작성
    - Spring이 @Configuration가 붙은 클래스를 싱글톤으로 생성하여 관리
        ~~~
        @Configuration
        public class AppConfig {

            @Bean
            public UserService userService() {
                return new UserService(orderService());
            }

            @Bean
            public ChannelService channelService() {
                return new OrderService();
            }
        }
        ~~~
    1) 장점
        - 어떤 객체가 Bean으로 등록 되는지 확인 가능
        - 외부 라이브러리 객체도 등록 가능
        - 테스트나 환경별 설정 분리 가능

    2) 단점
        - 클래스가 많아지면 설정 파일이 많아져 관리가 어려움
        - @Component 계열보다 설정이 복잡

3. @Conditional 계열 어노테이션
    - 특정 조건에 따라 Bean을 선택적으로 등록하는 방식

    1) 장점
        - 환경, 라이브러리 존재 여부, 속성 값에 따라 등록 가능
        - 개발, 테스트, 운영 환경에 따라 대응이 가능
        - 충돌 방지 및 기본값 제공

    2) 단점
        - 빈이 등록되지 않았을 때의 디버깅이 어려움
        - 조건이 많아지면 코드가 복잡해짐
