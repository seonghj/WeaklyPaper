## HashSet의 내부 동작 방식과 중복 제거 메커니즘을 설명하고, HashSet이 효율적인 중복 체크를 할 수 있는 이유를 설명해주세요.
1. 내부 동작 방식
    - HashMap을 사용하여 구현되어있음
        * 삽입하는 값을 Key값으로 사용
        * value에는 모든 요소가 똑같은 상수를 대입함 (즉, HashSet에서는 사용하지 않음)  

2. 중복 제거 메커니즘
    - HashMap에서 중복처리하는 것과 동일하게 작동
    - hashCode()와 equals() 두 메소드를 활용
        1) hashCode()
            - 객체를 정수 형태로 해싱하는 메서드
            - 같은 객체의 해시코드는 언제나 동일
        2) equals()
            - 두 객체의 Key값과 Value값이 일치하는지만 확인하는 메서드
            - HashSet에서는 모든 요소에 같은 Value를 저장하기 때문에 key값을 비교하여 같다면 주소가 달라도 같은 객체로 본다.
    - 두 메서드를 함께 쓰는 이유?
        * 해시 테이블 구조이기 때문
        * hashCode()로 같은 객체인지 판별, 해시 충돌이 일어난 경우 equals()로 값을 확인함

3. 효율적인 중복 체크를 할 수 있는 이유?
    1) 해시 값을 사용하여 데이터가 저장된 버킷을 빠르게 찾기 때문
        - 해시 충돌이 일어나지 않으면 O(1)로 객체를 바로 찾을 수 있음
        - 최악의 경우 = 같은 해시값에 연결리스트로 여러 값이 저장되는 경우이므로 O(n)

## O(n)과 O(log n)의 성능 차이를 실생활 예시를 들어 설명하고, 데이터의 크기가 1백만 개일 때 각각 대략 몇 번의 연산이 필요한지 비교해주세요.
1. O(n)
    - 걸리는 시간: 데이터를 하나하나 보기 때문에 늘어나는 만큼 정비례함
    - 예시: 정렬이 되어있지 않은 사전에서 단어를 찾는 경우 첫페이지 부터 하나하나 읽어서 검색
    - 데이터의 크기가 1백만 개일 경우: 최악의 경우 1백만 번째까지 작동

2. O(log n)
    - 걸리는 시간: 데이터를 볼 때마다 다음 후보가 절반씩 줄어들며 O(n)의 경우와 비교하여 더 빠름
    - 예시: 정렬이 되어있는 사전에서 중간을 펼쳐보고, 펼친 곳 기준으로 단어가 존재할 것으로 예상되는 곳에서 다시 중간을 펼침 (다른 곳은 버림)
    - 데이터의 크기가 1백만 개일 경우: 최악의 경우 20번 까지
